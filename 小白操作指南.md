# 家宽平台代理平台小白操作指南

这是一份给新手看的操作文档。

你不需要懂代码，也不需要懂接口，只要按下面的步骤做，就可以完成：

- 管理员登录后台
- 配置上游
- 给用户划拨额度
- 用户查询线路
- 用户先看报价
- 用户确认下单
- 查看订单和实例

## 三、这个平台的真实业务逻辑

这个平台不是在线支付模式。

正确流程是：

```text
用户联系你付款
↓
你线下确认收款
↓
管理员进入后台给这个用户划拨额度
↓
用户登录平台查询线路、获取报价、确认下单
↓
平台调用 IPIPD 官方接口创建真实订单
↓
用户拿到代理实例
```

也就是说：

- 用户自己不能直接充值
- 用户要先有额度，才能下单
- 管理员必须先给用户加额度

## 四、管理员第一次要做什么

管理员第一次进入后台后，先做下面两件事。

### 1. 检查上游凭据

进入左侧菜单：

```text
上游凭据
```

确认以下内容：

- 模式：`live`
- API Base URL：`https://api.ipipd.cn`
- 已填写 AppId
- 已填写 AppSecret

然后点击：

```text
查询上游账户
```

如果看到上游账户信息、余额等内容，说明已经成功连接 IPIPD 官方服务。

如果查询失败，先不要让用户下单，先检查上游凭据。

---

### 2. 同步真实线路价格

进入左侧菜单：

```text
价格管理
```

点击：

```text
同步上游线路
```

作用：

- 把 IPIPD 上游真实线路同步到本平台
- 生成可销售的 SKU
- 用户后面查询线路时，才能匹配到本地售价

同步成功后，页面里会出现很多 SKU。

如果没有同步线路，用户即使查到上游线路，也可能无法下单。

## 五、管理员如何创建用户

用户有两种方式进入平台：

### 方式 A：用户自己注册

用户打开首页后：

1. 点击“去注册”
2. 输入用户名和密码
3. 注册成功后自动登录

这是最简单的方式。

### 方式 B：管理员先让用户自己注册，再后台管理

管理员不需要手动建号。

用户注册完成后，管理员去后台：

```text
用户管理
```

就能看到这个用户。

## 六、管理员如何给用户划拨额度

这是最重要的后台操作。

用户付款后，管理员需要手动给用户加额度。

进入左侧菜单：

```text
额度划拨
```

页面会有一个表单。

按下面填写：

1. 用户：选择对应用户名
2. 金额：输入要加的额度
3. 线下付款方式：例如“银行转账”“支付宝”“USDT”
4. 凭证号：填写转账单号、流水号、备注号
5. 备注：写清楚原因，例如“客户已付款，补 180 额度”

然后点击：

```text
确认划拨
```

### 划拨金额怎么填

```text
100   = 给用户增加 100 额度
-50   = 从用户账户扣减 50 额度
```

### 划拨成功后会发生什么

- 用户余额会马上增加
- 系统会记录额度流水
- 用户重新刷新页面后就能看到余额变化

## 七、用户如何正常下单

用户登录后，正确的下单顺序是：

```text
先查线路
↓
选线路
↓
获取报价
↓
确认并下单
```

不能跳过“获取报价”这一步。

---

## 八、用户详细下单步骤

进入左侧菜单：

```text
下单
```

页面上有几个区域：

- 本地售价模板
- 数量
- 天数
- 业务类型
- 国家/地区
- 城市/可用区
- IP 类型
- 线路标签
- 线路 ID

### 第 1 步：先查询上游线路

先点击：

```text
查询上游线路
```

系统会从 IPIPD 官方拉真实线路。

下面会显示线路列表。

你要看两列：

- 库存
- 售价

如果某条线路显示：

```text
未配置售价
```

说明这条线路虽然上游有，但本平台还没给它配置销售价格，用户不能直接买。

如果显示具体价格，比如：

```text
￥14.00/CNY
```

说明这条线路可以下单。

### 第 2 步：选择一条可下单线路

在线路列表里找到一条想买的线路，点击：

```text
使用
```

点击后，系统会自动把：

- 线路 ID
- 业务类型
- 国家
- 城市
- IP 类型

带回到上面的表单里。

### 第 3 步：填写数量和天数

建议第一次测试：

```text
数量：1
天数：30
```

> 重要规则  
> 当前上游真实下单时，天数必须是 30 的倍数。  
> 所以不要填 1 天、7 天、15 天。  
> 正确示例：
>
> - 30 天
> - 60 天
> - 90 天

### 第 4 步：获取报价

点击：

```text
获取报价
```

系统会显示：

- 线路 / SKU
- 单价
- 数量 / 天数
- 总价
- 当前余额
- 下单后余额
- 业务类型
- 地区

### 第 5 步：检查报价

你需要看三件事：

1. 价格是不是对的
2. 线路是不是你要的
3. 余额够不够

如果余额不够，页面会提示额度不足。

这时候不要下单，先联系管理员划拨额度。

### 第 6 步：确认并下单

确认报价没问题后，点击：

```text
确认并下单
```

系统会：

1. 扣除用户余额
2. 调用 IPIPD 官方接口创建订单
3. 把上游返回的结果写回本平台

## 九、下单后去哪里看结果

下单成功后，用户可以看两个地方。

### 1. 我的订单

进入：

```text
我的订单
```

这里可以看到：

- 订单号
- SKU
- 数量和天数
- 金额
- 状态
- 上游订单号

常见状态：

```text
processing       处理中
completed        已完成
upstream_failed  上游失败
cancelled        已取消
```

### 2. 实例管理

进入：

```text
实例管理
```

这里会看到真实代理实例：

- Proxy ID
- IP
- 端口
- 用户名
- 密码
- 到期时间
- 状态

如果订单完成了，实例通常会出现在这里。

## 十、用户拿到实例后还能做什么

用户在“实例管理”里可以继续操作自己的代理：

- 批量换 IP
- 续费
- 更新代理账号密码

### 1. 换 IP

操作：

1. 勾选实例
2. 点击“批量换 IP”

### 2. 续费

操作：

1. 勾选实例
2. 输入续费天数
3. 点击“续费”

> 注意  
> 续费也会扣余额。  
> 余额不足时，要先让管理员划拨额度。

### 3. 更新账号密码

操作：

1. 勾选实例
2. 填写代理用户名和密码
3. 点击“更新账号密码”

如果不填用户名和密码，系统会请求上游随机生成。

## 十三、最容易出错的地方

这里是最常见的问题，新手一定要看。

### 1. 用户没额度就去下单

结果：

```text
余额不足
```

正确做法：

先让管理员在“额度划拨”里给用户加额度。

---

### 2. 用户没有先查线路，直接乱填参数

结果：

- 查不到真实线路
- 下单参数可能不匹配上游

正确做法：

先点“查询上游线路”，再点“使用”。

---

### 3. 用户没有先获取报价

结果：

无法确认价格和余额

正确做法：

一定要先点：

```text
获取报价
```

然后再点：

```text
确认并下单
```

---

### 4. 天数不是 30 的倍数

结果：

上游会拒单。

当前已经验证过，真实上游会返回：

```text
天数必须是30的倍数
```

正确做法：

只填：

- 30
- 60
- 90

---

### 5. 同步过线路，但用户看到“未配置售价”

这说明：

- 上游有这条线路
- 但本平台还没给这条线路配置出可售价格

正确做法：

管理员进入“价格管理”检查对应 SKU 是否存在、是否启用、是否有销售价。

---

### 6. 下单失败但余额被扣了怎么办

系统已经做了保护。

如果上游失败，系统会自动退回额度，并把订单状态标记成：

```text
upstream_failed
```

管理员可以去：

- 订单管理
- 额度流水

查看具体情况。

---

## 十四、推荐的新手操作顺序

如果你第一次正式用，建议按这个顺序：

### 管理员

1. 登录后台
2. 检查“上游凭据”是否为 `live`
3. 点击“查询上游账户”
4. 去“价格管理”同步上游线路
5. 让用户先注册
6. 去“额度划拨”给用户加额度

### 用户

1. 登录平台
2. 去“下单”
3. 点击“查询上游线路”
4. 选择一条可下单线路，点击“使用”
5. 填数量和天数（建议 1 个、30 天）
6. 点击“获取报价”
7. 看价格和余额
8. 点击“确认并下单”
9. 去“我的订单”看状态
10. 去“实例管理”看代理信息

---

## 十五、如果出现问题，先看哪里

### 管理员先检查

1. 上游凭据是不是 `live`
2. 上游账户是否能查询成功
3. 价格是否已经同步
4. 用户是否有额度
5. 天数是否为 30 的倍数

### 用户先检查

1. 是否已经登录
2. 是否先查询线路
3. 是否先获取报价
4. 余额是否足够
5. 天数是否填了 30 / 60 / 90

```text

```
