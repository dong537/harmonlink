## 二、用户端使用说明

用户端包含以下功能：

- 概览
- 下单
- 实例管理
- 我的订单
- 额度记录

### 1. 注册与登录

打开首页后，用户可以选择注册或登录。

注册流程：

1. 点击“去注册”。
2. 输入用户名和密码。
3. 注册成功后自动登录。

登录流程：

1. 输入用户名。
2. 输入密码。
3. 点击“登录”。

### 2. 查看余额和价格

进入“概览”页面后，可以查看：

- 当前账户余额
- 订单数量
- 额度流水数量
- 可购买 SKU
- 每个 SKU 的实际单价

如果管理员为当前用户设置了 SKU 覆盖价，则用户看到的是覆盖后的价格。

### 3. 线下付款与额度到账

当前平台不接入在线支付，也不由用户提交充值申请。用户完成线下付款后，需要联系平台管理员确认收款。

进入“额度记录”页面可以查看：

- 管理员划拨的额度
- 扣减记录
- 划拨前余额
- 划拨后余额
- 付款方式或凭证备注

管理员确认线下付款后，会在管理后台为用户划拨对应额度。额度到账后，用户即可使用余额下单。

### 4. 下单购买家宽 IP

进入“下单”页面：

1. 选择 SKU。
2. 填写数量。
3. 填写购买天数。
4. 确认业务类型、国家/地区、IP 类型、线路标签。
5. 可选填写线路 ID。
6. 点击“提交订单”。

系统会先根据用户实际价格计算订单金额并扣除余额，然后调用 IPIPD 上游接口创建订单。

余额不足时无法下单，需要先线下付款并由管理员划拨额度。

### 5. 查询上游线路

在“下单”页面点击“查询上游线路”，系统会调用 IPIPD 的线路接口：

```text
/openapi/v2/static/lines
```

该功能用于查看当前筛选条件下的可用线路。

### 6. 查看订单

进入“我的订单”页面可以查看：

- 本地订单号
- SKU
- 数量和天数
- 金额
- 状态
- 上游订单号或错误信息

常见订单状态：

```text
processing        处理中
completed         已完成
upstream_failed   上游创建失败
cancelled         已取消
```

### 7. 管理代理实例

下单成功后，IPIPD 返回的实例会显示在“实例管理”页面。

可以查看：

- Proxy ID
- IP
- 端口
- 代理用户名
- 代理密码
- 地区
- IP 类型
- 到期时间
- 状态

支持操作：

- 批量换 IP
- 续费
- 更新代理账号密码

操作前需要先勾选实例。

### 8. 批量换 IP

在“实例管理”页面：

1. 勾选实例。
2. 点击“批量换 IP”。

系统会调用上游接口：

```text
/openapi/v2/static/instances/change-ip
```

### 9. 续费实例

在“实例管理”页面：

1. 勾选实例。
2. 输入续费天数。
3. 点击“续费”。

系统会根据原订单单价扣除用户余额，并调用上游接口：

```text
/openapi/v2/static/instances/renew
```

### 10. 更新代理账号密码

在“实例管理”页面：

1. 勾选实例。
2. 可填写代理用户名和代理密码。
3. 点击“更新账号密码”。

如果用户名和密码都留空，系统会请求上游随机生成。

对应上游接口：

```text
/openapi/v2/static/instances/update-credentials
```

## 三、管理员端使用说明

管理员登录后进入管理后台。

后台包含以下功能：

- 管理概览
- 上游凭据
- 用户管理
- 价格管理
- 订单管理
- 实例管理
- 额度划拨

### 1. 管理概览

管理概览显示：

- 用户数
- 订单数
- 额度流水数量
- 启用 SKU 数量

### 2. 配置 IPIPD 上游凭据

进入“上游凭据”页面。

字段说明：

```text
模式：mock 或 live
API Base URL：https://api.ipipd.cn
AppId：IPIPD 应用 ID
AppSecret：IPIPD 应用密钥
平台账号：IPIPD 登录账号，可记录备用
平台密码：IPIPD 登录密码，可记录备用
```

默认模式为：

```text
mock
```

mock 模式不会真实请求 IPIPD，适合测试本地业务流程。

正式对接时：

1. 填写 AppId。
2. 填写 AppSecret。
3. 将模式切换为 `live`。
4. 点击保存。
5. 点击“查询上游账户”测试连接。

系统按 IPIPD 文档使用 HMAC-SHA256 签名，请求头包括：

```text
X-API-AppId
X-API-Timestamp
X-API-Nonce
X-API-Signature
```

### 3. 用户管理

进入“用户管理”页面可以：

- 查看用户
- 修改密码
- 加款或扣款
- 启用或禁用账号
- 设置用户 SKU 覆盖价格

#### 修改密码

在用户行输入新密码，点击“保存”。

#### 调整余额

在“加减款”输入金额：

```text
100     表示加 100
-50     表示扣 50
```

然后点击“保存”。

#### 禁用用户

点击“禁用”。

禁用后该用户无法登录。

#### 设置用户价格覆盖

每个用户可以针对某个 SKU 设置独立价格。

操作流程：

1. 在用户行选择 SKU。
2. 输入覆盖价。
3. 点击“设置”。

设置后，该用户下单时按覆盖价计算，不影响其他用户。

### 4. 价格管理

进入“价格管理”页面可以管理 SKU。

SKU 由以下字段组成：

```text
业务类型
国家/地区代码
城市/可用区代码
IP 类型
线路标签
```

系统内部 SKU 格式：

```text
businessType|countryCode|cityCode|ispType|tag
```

示例：

```text
WEB|USA|*|1|STANDARD
```

含义：

```text
WEB 业务
美国
不限城市
广播 IP
STANDARD 标签
```

#### 新增 SKU

填写：

- 业务类型
- 国家代码
- 可用区或城市
- IP 类型
- 标签
- 名称
- 上游价/天
- 销售价/天
- 币种

点击“保存 SKU”。

#### 修改 SKU 价格

在表格中修改：

- 名称
- 上游价
- 销售价

然后点击“保存”。

#### 同步上游线路

点击“同步上游线路”。

系统会调用：

```text
/openapi/v2/static/lines
```

并把返回线路转换为本地 SKU。

### 5. 订单管理

进入“订单管理”页面可以查看全部订单。

可以查看：

- 订单号
- 用户
- SKU
- 金额
- 状态
- 备注

管理员可以手动调整订单状态和备注。

### 6. 实例管理

进入“实例管理”页面可以查看所有用户的代理实例。

可以查看：

- Proxy ID
- 所属用户
- 所属订单
- IP
- 端口
- 代理账号
- 代理密码
- 到期时间
- 状态

### 7. 额度划拨

进入“额度划拨”页面可以在用户线下付款后，为用户账号增加对应额度，也可以在需要时扣减额度。

划拨额度：

1. 选择用户。
2. 输入金额。
3. 填写线下付款方式。
4. 填写付款凭证号或备注。
5. 点击“确认划拨”。

金额说明：

```text
100     表示给用户增加 100 额度
-50     表示从用户余额扣减 50 额度
```

系统会记录完整额度流水，包括操作人、付款方式、凭证号、划拨前余额和划拨后余额。

## 四、IPIPD 回调配置

系统提供回调地址：

```text
http://你的域名/api/ipipd/callback
```

正式环境必须使用 HTTPS，例如：

```text
https://your-domain.com/api/ipipd/callback
```

支持 IPIPD 文档中的回调类型：

```text
CREATE_STATIC_ORDER
UPDATE_STATIC_INSTANCE
```

### 1. CREATE_STATIC_ORDER

用于上游通知订单创建成功。

系统会：

- 匹配本地订单
- 更新订单状态
- 写入实例列表

### 2. UPDATE_STATIC_INSTANCE

用于上游通知实例变更。

例如：

- IP_CHANGE
- EXPIRY
- RENEWAL
- CREDENTIALS_CHANGE

系统会更新本地实例的：

- IP
- 端口
- 用户名
- 密码
- 状态
- 到期时间

## 五、数据与安全说明

当前系统使用本地 JSON 文件存储数据：

```text
data/db.json
```

适合演示、测试、小规模内部使用。

正式生产建议后续升级为：

- MySQL 或 PostgreSQL
- Redis Session 或更完整的 JWT 密钥管理
- HTTPS
- Nginx 反向代理
- 管理员密码强制修改
- 上游 AppSecret 加密存储
- 操作日志
- 财务流水表

## 六、常见问题

### 1. 余额不足无法下单

用户需要先线下付款，管理员确认收款后在“额度划拨”页面为用户增加额度，然后用户再下单。

### 2. 上游请求失败

检查后台“上游凭据”：

- 是否切换到 `live`
- AppId 是否正确
- AppSecret 是否正确
- API Base URL 是否为 `https://api.ipipd.cn`

也可以切回 `mock` 模式验证本地业务流程。

### 3. 用户看到的价格和后台价格不同

说明管理员在“用户管理”里给该用户设置了 SKU 覆盖价。

### 4. 下单成功但没有实例

可能原因：

- 上游订单异步处理
- 上游还未回调
- 回调地址未配置或不可访问

可以等待 IPIPD 回调，或在管理后台查看订单上游返回内容。
